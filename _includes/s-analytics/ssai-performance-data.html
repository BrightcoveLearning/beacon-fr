Présentation

Brightcove Server-Side Ad Insertion (SSAI) intègre des publicités à votre flux de contenu pendant sa diffusion. Comme l'assemblage de publicités se fait côté serveur, SSAI fonctionne avec n'importe quelle plateforme. Pour plus d'informations sur SSAI, consultez le document Video Cloud SSAI Overview.

Les analyses de performance SSAI sont prises en charge avec les éléments suivants:

  • VOD et diffusions en direct
  • Le lecteur Web de Brightcove, les SDK natifs et les applications Beacon
  • Video Cloud et ressources distantes

Dans {{site.product_short}} Studio, dans le module Analytics, la section SSAI Performance vous permet de surveiller l'efficacité de vos fournisseurs de publicités et de leurs configurations publicitaires. Vous pouvez y suivre les performances et les erreurs associées à votre publicité côté serveur.

Il peut y avoir une certaine latence de 30 à 60 minutes avant que vous ne voyiez les données dans l'interface utilisateur de Studio. Dans certains cas, l'affichage des données SSAI Performance peut prendre jusqu'à une journée.

Configuration

Cette fonctionnalité est disponible pour tous les clients qui ont activé SSAI pour leur compte. Contactez votre responsable de compte pour les informations suivantes:

  • Assurez-vous que votre compte est activé pour Brightcove Server-Side Ad Insertion (SSAI).
  • Activez votre compte pour les analyses de performance SSAI.

Flux de travail

Suivez ce flux de travail pour commencer à voir les données d'analyse associées à SSAI:

  1. Créez une configuration publicitaire (AdConfig) avec le suivi activé. Pour ce faire, vous allez inclure le paramètre ad_tracking_sample_percentage dans la configuration.

    • Pour la VOD, consultez le document de l'API SSAI Video Cloud.
    • Pour les diffusions en direct, consultez la référence Créer une configuration publicitaire.
    si vous créez un nouvel AdConfig, vous devrez activer le suivi.
  2. Associez la configuration de la publicité à votre lecteur.
    • Pour le lecteur Web Brightcove, consultez le document Implementing Server-Side Ads with Brightcove Player.
    • Pour les SDK natifs, consultez le document Implémentation des publicités côté serveur avec les SDK natifs.
    • Pour les applications Brightcove Beacon, consultez les rubriques relatives aux SSAI.

Performances SSAI

La page Performances SSAI fournit un résumé des demandes d'annonces, à la fois des réussites et des erreurs.

Plage de dates

Utilisez le menu déroulant pour sélectionner une plage de dates prédéfinie ou entrez une date ou une plage de dates personnalisée.

Vues des données

Les données de cette page indiquent les codes de réponse de chaque demande envoyée à votre serveur publicitaire. Il existe deux points de communication avec le serveur publicitaire:

  • Ad Request: lorsque le lecteur fait une demande d'annonce
  • Ad Tracker: lorsque des événements de suivi publicitaire sont déclenchés à partir du serveur SSAI de Brightcove ; il peut s'agir des éléments suivants:
    • AdStart
    • Annonce complète
    • Premier quartile de l'annonce
    • Point d'admission
    • Troisième quartile
    • Vue créative d'AD

La page est divisée en cinq sections de données:

  • Codes de demande SSAI: tous les codes de réponse de chaque demande envoyée à votre serveur publicitaire, y compris les codes de réussite et d'erreur
  • Erreurs de demande d'annonce par fournisseur d'annonces: liste des erreurs de demande triées par fournisseur d'annonces
  • Erreurs de demande d'annonce par configuration publicitaire: liste des erreurs de demande triées par configuration publicitaire
  • Erreurs de suivi par fournisseur d'annonces: liste des erreurs de suivi triées par fournisseur d'annonces
  • Erreurs de suivi par configuration publicitaire: liste des erreurs de suivi triées par configuration publicitaire

Cliquez sur Afficher plus pour plus de détails.

Affichage des données

Voici le détail des données affichées sur cette page:
