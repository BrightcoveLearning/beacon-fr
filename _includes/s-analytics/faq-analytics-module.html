1. Qu'est-ce que le module Analytics ?

Le module Analytics est un module d'interface utilisateur que vous pouvez utiliser pour afficher des informations d'analyse en temps quasi réel sur vos vidéos et vos lecteurs. La plate-forme Analytics utilise les dernières innovations en matière de technologie Big Data et est conçue pour une grande échelle. La plateforme d'analyse offre un accès rapide et quasiment en temps réel aux données d'analyse et inclut une puissante API ouverte qui permet aux développeurs de créer des solutions et des rapports d'analyse personnalisés en plus de ceux inclus dans {{site.product_short}} Studio.

2. Puis-je exporter des données de rapport à partir du module d'analyse ?

Oui. Les rapports d'analyse contiennent des liens qui peuvent être utilisés pour télécharger les données du rapport dans des fichiers .csv et .xlsx.

3. À quelle fréquence les données sont-elles mises à jour dans le module d'analyse ?

Normalement, les données proches du temps réel sont retardées d'environ 15 minutes. Ce délai peut toutefois varier en fonction du volume de données actuellement reçu par le système de collecte de données et, dans certains cas, le délai peut atteindre 3 heures.

Notez que le système Analytics compile des données provenant de sources multiples, et que les données proches du temps réel sont une approximation basée sur les données collectées jusqu'à présent. Les chiffres peuvent varier d'un rapport à l'autre et doivent être considérés comme des tendances plutôt que des chiffres précis.

4. Existe-t-il un moyen d'inclure nos identifiants vidéo internes dans les rapports ou dans votre API ?

Il existe un champ ID de référence disponible en tant que propriété vidéo que vous pouvez utiliser pour mapper votre identifiant interne aux vidéos dans {{site.product_short}}. Le champ ID de référence peut être affiché lors de la création d'un rapport personnalisé. Pour plus d'informations sur les rapports personnalisés, consultez Création de rapports Analytics personnalisés dans le module Analytics.

5. Que se passe-t-il s'il y a une publicité pré-roll avant une vidéo et qu'un spectateur regarde l'annonce pendant quelques secondes, puis s'en va ? Si le spectateur n'a pas du tout regardé le contenu principal, cela compte-t-il comme un visionnage de vidéo ?

Non. Une vidéo visionnée n'est prise en compte qu'après le début de la lecteur du contenu principal.

6. Est-il possible de filtrer les plages d'adresses IP de nos analyses ?

La possibilité de filtrer les plages d'adresses IP n'est pas disponible pour le moment.

7. Est-ce que Analytics suit les lecteur s grâce à la diffusion d'événements en direct

Analytics suit les minutes consultées dans le cadre du rapport sur les performances. Les secondes visionnées en direct peuvent être affichées lors de la création de rapports personnalisés ou via le module Facturation. La bande passante des événements en direct n'est pas affichée

8. Comment puis-je trouver des informations sur une vidéo spécifique via le module d'analyse ?

La fonction de recherche peut être utilisée pour rechercher une vidéo spécifique. Le nom de la vidéo, la description courte, la description longue, les balises, l'identifiant de la vidéo, l'ID de référence et tous les champs de texte personnalisés dont vous disposez seront recherchés. Cliquez sur un résultat pour afficher la performance ou l'engagement de cette vidéo. Des rapports personnalisés peuvent également être créés pour afficher les analyses pour un identifiant vidéo donné.

9. Puis-je rechercher une vidéo spécifique à l'aide de l'ID de référence dans le module d'analyse ?

Oui. La fonction de recherche peut être utilisée pour rechercher une vidéo sur la base du champ ID de référence. Le nom de la vidéo, la description courte, la description longue, les balises, l'identifiant de la vidéo et tous les champs de texte personnalisés dont vous disposez seront également recherchés.

10. Comment pouvez-vous rendre compte de l'engagement des clients et de déposer une vidéo en particulier ?

Le rapport d'engagement présente ces informations. Lorsque vous sélectionnez le rapport d'engagement et que vous sélectionnez la dimension Vidéo, vous verrez un graphique montrant l'engagement global pour votre compte. Si vous sélectionnez une vidéo, vous verrez un graphique affichant l'engagement pour cette vidéo au fil du temps.

11. Puis-je créer un rapport sur l'URL ? Par exemple, si nous avons la même vidéo hébergée sur la page A et la page B, pouvons-nous comparer les performances entre les deux emplacements ?

Oui. Le rapport de performance offre la possibilité de visualiser les analyses vidéo par domaine d'affichage.

12. Si une vidéo est en boucle, est-ce que cela compte pour une ou plusieurs vues à chaque lecteur de la vidéo ?

Les visionnages de vidéos ne sont comptés qu'une fois par session de visionnage au démarrage du flux Notez que le nombre de vues à x% dans le rapport d'engagement n'est pas identique à celui d'une vue vidéo dans le rapport de performance. Une vue vidéo est prise en compte lorsqu'une vidéo commence à être lue pour la première fois. Les vues à 1 % sont enregistrées lorsque le premier pourcentage est regardé. La différence ici est que si le premier pourcentage est regardé plusieurs fois (si la vidéo est configurée pour être rejouée automatiquement, par exemple), le nombre augmente, tandis qu'une vidéo vue est comptée lorsque la vidéo commence à être lue pour la première fois, mais n'est pas compté sur une rediffusion (sauf si l'utilisateur recharge la page ou regarde une autre vidéo dans la liste de lecteur, etc.).

13. Puis-je générer un rapport pour afficher le nombre de spectateurs uniques pour mon compte ?

Le nombre de spectateurs uniques est indiqué dans le rapport de performance.

14. Les rapports standard disponibles dans le module Analytics peuvent-ils être personnalisés ?

Les données affichées dans les rapports Performance et Engagement ne peuvent pas être personnalisées. Cependant, le module Analytics permet de créer des rapports d'analyse personnalisés vous permettant de sélectionner les données à afficher. Il existe également une API Analytics qui peut être utilisée pour accéder aux données d'analyse par programme.

15. Pourquoi certains scores d'engagement sont-ils très élevés, supérieurs à 100 ?

Il est possible que le score d'engagement soit supérieur à 100. Le nettoyage et le visionnage d'une vidéo ne déclenchent pas de nouveaux événements video_view, mais suivent video_engagement. Par exemple, supposons que nous ayons une vidéo dans un seul lecteur vidéo sur une page quelque part. Les événements suivants montrent comment l'engagement vidéo peut varier considérablement:

  1. Pas encore de montres. Score d'engagement: 0.
  2. Un spectateur charge la page et regarde la vidéo en entier. Score d'engagement: 100 (100 % visualisés/1 vue).
  3. Le même spectateur revient en arrière et le regarde à nouveau (le navigateur n'a pas été rechargé). Score d'engagement: 200 (200 % visualisés/1 vue).
  4. Un autre utilisateur charge la page et regarde la vidéo en entier. Score d'engagement: 150 (300 pour cent vues/2 vues).

Pour plus de détails sur la façon dont nous calculons l'engagement vidéo, regardez la vidéo de formation Calculer l'engagement vidéo.

16. Pourquoi la mesure des vues à 25 % est-elle supérieure à celle des vues à 1 % ?

Deux facteurs peuvent être à l'origine de l'écart entre les deux chiffres:

  • Les vues à 1 % et les vues à x % sont deux mesures différentes. Le nombre de vues à 1 % est unique par session de visionnage et n'inclut pas le retour en arrière et le replay. Lorsqu'un spectateur rejoue ou revient à une partie précédente d'une vidéo, le point de pourcentage augmentera le nombre d'audience, mais pas le nombre de vues de la vidéo.
  • Les vues en % et la courbe d'engagement sont stockées dans une granularité quotidienne en utilisant l'heure UTC, de sorte qu'une date spécifique peut parfois inclure 2 jours de données si l'UTC et le fuseau horaire de reporting se croisent.

17. Pourquoi le nombre de visionnages de vidéos pour une journée donnée varie-t-il au fil du temps ?

Le système d'analyse fournit des mesures récentes non rapprochées collectées chaque heure et se réconcilie automatiquement avec les mesures réelles signalées au niveau quotidien. Ce rapprochement a lieu dès que possible, généralement en 3 à 5 jours. Nous ne réconcilions jamais les données collectées au niveau horaire. Puisque les éditeurs valorisent les données horaires, nous mettons les données non rapprochées horaires à disposition pendant 32 jours. Quelques détails supplémentaires sur le rapprochement:

  • Les lecteur s envoient des données à l'API de collecte de données
  •  L'API de collecte de données fait de son mieux pour écrire dans la base de données en temps réel ET elle écrit les événements dans le stockage des journaux
  • Au fur et à mesure que nous collectons des données supplémentaires à partir de CDN et d'autres sources, les données analytiques sont mises à jour ; il s'agit du processus
  • Un système de traitement par lots nocturne examine les événements du journal brut et crée des fichiers d'importation de données historiques
  • Les fichiers d'importation de données sont chargés dans la base de données historique. Lorsque vous effectuez un appel d'API, vous pouvez choisir de ne voir que les données rapprochées (plus précises) ou de voir à la fois les données rapprochées et non rapprochées à l'aide du paramètre réconcilié. Consultez la présentation de l'API Analytics pour plus d'informations.
  • Toutes les données historiques sont rapprochées ; les données en temps réel (disponibles pendant 32 jours) peuvent avoir été rapprochées ou non. Si la période pour laquelle vous obtenez des données se termine il y a plus de 32 jours, il n'est pas nécessaire d'utiliser les données rapprochées, car les données seront toutes historiques.

18. Pourquoi certaines mesures renvoient-elles une valeur nulle ou n'ont-elles aucune valeur ?

Cela se produit si aucune mesure n'est disponible pour la période que vous avez demandée. Certaines mesures ont une latence de 2 à 3 jours. Par conséquent, si, par exemple, vous avez demandé ces données au cours des 30 derniers jours (y compris aujourd'hui), elles peuvent être nulles puisque la plage demandée inclut des jours pour lesquels ces données ne sont pas encore disponibles. Pour éviter cela lors de la demande de données, assurez-vous que la plage demandée se situe en dehors de la période de latence.

Nous avons également ajouté des mesures et des combinaisons de dimensions supplémentaires au fil du temps. Ainsi, par exemple, si vous exécutez un rapport pluriannuel, certaines mesures peuvent être vides ou partielles, car nous n'avons pas suivi cette mesure pendant toute la période de reporting. Pour plus d'informations sur les dates valides et les combinaisons de dimensions, consultez le document Analytics API Overview.

19. Pourquoi les données du domaine de destination de 2013 ne correspondent-elles pas aux autres données de performance ?

La fonctionnalité de domaine de destination n'a été activée qu'en janvier 2014. Les données de mars 2013 à janvier 2014 ont été partiellement remplies, mais seules certaines données de destination sont disponibles pour cette période. Les données de janvier 2014 sont complètes.

20. Les analyses pour les vidéos supprimées sont-elles incluses ?

Les rapports Analytics incluent des analyses pour les vidéos supprimées de votre compte. Toutefois, vous ne pouvez pas rechercher de vidéos supprimées dans le module Analytics pour afficher un rapport sur une vidéo supprimée en particulier. Pour obtenir un tel rapport, vous devez connaître l'identifiant de la vidéo supprimée et utiliser l'API Analytics pour générer le rapport. Notez que si les données du rapport sont exportées, les données des vidéos supprimées peuvent apparaître sans afficher d'identifiant ou de nom de vidéo.

21. Puis-je exclure les points de vue des employés des rapports d'analyse ?

Il n'y a pas de moyen simple de le faire, et pour les vidéos à fort trafic, le pourcentage de vues des employés est probablement si faible que leur effet sur les analyses est négligeable. Toutefois, pour les vidéos à faible trafic pour lesquelles vous pensez que c'est important de le faire, la solution la plus simple consiste à dupliquer votre/vos lecteur (s) de production et à demander aux employés de tester/visionner les vidéos sur les copies. Vous pouvez ensuite utiliser des filtres pour créer des rapports sur votre/vos lecteur (s) de production uniquement, à l'aide de la fonctionnalité Rapports personnalisés du module Analytics ou de l'API Analytics.

22. Pourquoi est-ce que je ne vois pas toujours les chiffres entiers pour l'engagement ?

Il est possible que les numéros d'engagement soient renvoyés avec des décimales. La raison en est que l'engagement est normalisé, ce qui signifie qu'il s'agit d'un ratio de video_percent_viewed * (video_engagement_25/video_engagement_sum)-fr/. Ainsi, en fonction de la plage temporelle sélectionnée pour la requête, vous pouvez voir des nombres à virgule flottante dans les cas où ils ne se divisent pas exactement.™

23. Pourquoi certains rapports affichent-ils uniquement l'identifiant de la vidéo, ou pourquoi certaines vidéos n'apparaissent-elles pas du tout ?

S'il n'y a pas de visionnage d'une vidéo pendant une période donnée, le rapport ne contiendra pas de données détaillées à ce sujet (par exemple, uniquement l'identifiant de la vidéo et rien d'autre). Si aucune donnée n'est téléchargée (c'est-à-dire pas même des données d'image), la vidéo n'apparaîtra pas dans le rapport.

24. Quelle est la première date à laquelle je peux obtenir des données d'analyse ?

La date la plus rapprochée pour toutes les données d'analyse est le 28 décembre 2010. Certains éléments de données n'ont été collectés que plus tard, de sorte que la date la plus rapprochée à laquelle vous pouvez voir les données dépendra exactement de ce que vous regardez. Les visionnages de vidéos sont disponibles depuis le 28 décembre 2010. Notez que lorsque vous parcourez des dates à l'aide du contrôle de calendrier, il revient uniquement à janvier 2013, mais les dates antérieures peuvent être saisies manuellement.

25. Le module Analytics et l'API Analytics doivent-ils me fournir des données identiques pour la même plage de dates ?

Oui, mais elles peuvent être différentes si vous comparez des données relativement récentes et que vous obtenez des données de Studio et de l'API à des moments différents:

  1. Les données datent de moins de 3 jours. Les données d'analyse datant de moins de 3 jours sont provisoires et peuvent être modifiées à tout moment.
  2. Les données pour la même période sont examinées lorsqu'il avait moins de 32 jours, puis à nouveau après 32 jours. Après 32 jours, les données sont déplacées vers un référentiel historique-fr/ distinct, et à ce stade, nous ignorons certaines données, en particulier les données d'engagement détaillées. Cela signifie que les champs calculés tels que le score d'engagement peuvent légèrement changer, car le calcul est effectué sur des données moins granulaires.

26. Pourquoi le nombre de vues de mes vidéos dans le module Analytics est-il supérieur au nombre de vues du module Campaign ?

Les modules Analytics & Campaign sont deux systèmes distincts et il ne faut pas s'attendre à ce que les événements de vue soient 1:1, car chacun capture des cas d'utilisation distincts.

Le formulaire de prospect de campagne nécessite le déclenchement d'un événement video_view avant de pouvoir afficher un formulaire de prospect. Ces événements video_view se déclenchent lorsqu'un utilisateur lance la lecteur, ce qui se fait en cliquant manuellement sur le bouton Lire. Par conséquent, même s'il peut sembler qu'aucun contenu n'a été lu, une video_view a été enregistrée et apparaîtra pour votre vidéo dans le module Analytics.

Si vous constatez un plus grand nombre de visionnages de vidéos dans le module Analytics que dans le module Campaign, la raison la plus probable est qu'un utilisateur s'est vu présenter votre formulaire de prospect, mais qu'il ne l'a pas rempli pour poursuivre la lecteur.
