Les données d'analyse sont envoyées automatiquement par les lecteurs Brightcove, y compris celles fournies par les SDK Native Player. Si vous n'utilisez pas de lecteur Brightcove pour diffuser {{site.product_short}} vidéos, vous devez instrumenter le lecteur que vous utilisez pour envoyer les données au collecteur de données. Voir la vue d'ensemble: Rubrique API de collecte de données pour plus de détails.

Pour afficher le rapport Performance, ouvrez le module Analytics et sélectionnez Performances dans le menu de navigation de gauche. Le rapport affichera un graphique montrant le nombre total de visionnages de vidéos par jour pour la période sélectionnée. La liste déroulante peut être utilisée pour sélectionner une autre mesure:

  • Vues de la vidéo: nombre de fois qu'une vidéo a commencé à être lue, enregistré au démarrage du flux (n'inclut pas le retour en arrière ou la rediffusion) ; il ne s'agit pas d'une mesure des spectateurs individuels ; pour les vidéos comportant des publicités pre-roll, une vue n'est pas enregistrée tant que la vidéo n'a pas commencé à être lue
  • Impressions vidéo: nombre de fois qu'une vidéo a été chargée et prête à être utilisée
  • Demandes de lecteur: nombre de fois où la lecteur a été demandée. Les demandes de lecteur sont générées par:
    • le spectateur clique initialement sur le bouton de lecteur
    • le spectateur interrompt la vidéo et clique sur Play pour reprendre la lecteur
    • un événement de lecteur automatique
    • le spectateur frotte ou clique sur la chronologie de lecteur (ce qui interrompt et redémarre automatiquement la vidéo)
  • Taux de lecteur: pourcentage de vidéos qui ont été chargées et qui ont été réellement lues (visionnages/impressions vidéo)
  • Viewed Minutes: nombre de minutes pendant lesquelles la vidéo a été visionnée
  • Score d'engagement: pourcentage moyen regardé par vidéo vue (calculé en additionnant tout le pourcentage regardé divisé par le nombre de vues de la vidéo)
  • Visionneuses uniques quotidiennes - Calculé en utilisant l'adresse IP et l'agent utilisateur des spectateurs. Brightcove a décidé de ne pas utiliser de cookie tiers car les cookies sont de plus en plus bloqués par les navigateurs. Daily Unique Viewers est disponible au niveau du compte ou de la vidéo ; cette mesure n'est pas disponible lors du filtrage par vidéo (ou autre). Brightcove permet également de baser les utilisateurs uniques sur la valeur que vous offrez au lieu de leur adresse IP. Pour plus d'informations, consultez le document destiné aux développeurs Analytics Event Tracking.
  • Bande passante vidéo: inclut la bande passante pour les vidéos uniquement. Notez que les données de bande passante sont retardées de 3 à 5 jours.
  • Bande passante totale: inclut la bande passante pour les vidéos, les images et les charges de lecteur. Notez que les données de bande passante sont retardées de 3 à 5 jours.

Par défaut, le rapport affiche les données de performance pour tous les types de flux. Si votre compte est activé pour Brightcove Live, vous pouvez choisir d'afficher les données de performance de toutes les vidéos, ressources VOD ou ressources en direct. Cliquez sur la pastille Stream type pour faire une sélection.

Remarques:
  • Les rapports Analytics incluent des analyses pour les vidéos supprimées de votre compte. Vous ne pouvez pas rechercher de vidéos supprimées dans le module Analytics pour voir un rapport sur une vidéo supprimée en particulier. Pour obtenir un tel rapport, vous devez connaître l'ID de la vidéo supprimée et utiliser l'API Analytics pour générer le rapport.
  • Il faut jusqu'à 3 jours pour que les données sur les spectateurs uniques quotidiens et les données de bande passante vidéo soient disponibles.
  • Le visionnage d'une vidéo n'est pas le même que le nombre de vues à x% indiqué dans le rapport d'engagement. Une vue vidéo est prise en compte lorsqu'une vidéo commence à être lue pour la première fois. Les vues à 1 % sont enregistrées lorsque le premier pourcentage est regardé. La différence ici est que si le premier pourcentage est regardé plusieurs fois (si la vidéo est configurée pour être rejouée automatiquement, par exemple), le nombre augmente, tandis qu'une vidéo vue est comptée lorsque la vidéo commence à être lue pour la première fois, mais n'est pas compté sur une rediffusion (sauf si l'utilisateur recharge la page ou regarde une autre vidéo dans la liste de lecteur, etc.).
  • Les demandes de rapport simultanées au sein d'un même compte seront exécutées en série. Si les rapports comportent de grandes plages de dates et que de nombreux champs sont sélectionnés, il peut y avoir un délai avant que tous les résultats ne soient renvoyés. Une bonne alternative consiste à planifier des rapports volumineux.
  • Avec les expériences In-Page, même si la ou les vidéos semblent être chargées dans un lecteur, les vidéos ne sont chargées que lorsque le spectateur clique sur le bouton de lecteur superposé. Par conséquent, les impressions vidéo ne seront pas enregistrées par le module Analytics tant qu'un spectateur n'aura pas lu une vidéo.

Comparaison avec la période précédente

Activez la case à cocher Comparer à la période précédente pour ajouter un graphique en pointillés superposé représentant les données de la période précédente.

La période précédente est basée sur ce qui est actuellement sélectionné pour la plage de dates du rapport. Par exemple, si vous sélectionnez une plage de dates des 7 derniers jours, la période précédente sera celle d'il y a 2 semaines. Lorsque vous comparez les tendances sur plusieurs semaines, il est préférable de sélectionner des plages de dates qui couvrent des semaines complètes. Par exemple, si vous sélectionnez une date commençant un lundi, définissez la date de fin sur un dimanche. Si la date de début est un vendredi, définissez la date de fin sur un jeudi. Vous pouvez également passer la souris sur les barres et la ligne pointillée pour voir la date et les valeurs correspondantes.

Détails du rapport de performance

La première colonne qui apparaît dans le corps du rapport varie en fonction de la dimension de rapport sélectionnée (nom de la vidéo, nom du lecteur , pays, type d'appareil, etc.). Tous les rapports affichent les données suivantes:

  • Vues vidéo: nombre de fois qu'une vidéo a commencé à être lue, enregistré au démarrage du flux (n'inclut pas le retour en arrière ou la rediffusion) ; il ne s'agit pas d'une mesure des spectateurs individuels
  • Impressions vidéo: nombre de fois où la vidéo a été chargée dans un lecteur et prête à être utilisée
  • Demandes de lecteur: nombre de fois où la lecteur a été demandée par un utilisateur qui a appuyé sur le bouton de lecteur ou par un événement de lecteur automatique
  • Taux de lecteur: pourcentage d'utilisateurs qui ont lu la vidéo après son chargement (nombre de vues de la vidéo divisé par le nombre d'impressions vidéo)
  • Viewed Minutes: nombre total de minutes pendant lesquelles la vidéo a été visionnée
  • Score d'engagement: pourcentage moyen regardé par vidéo visionnée (calculé en additionnant tout le pourcentage regardé divisé par le nombre de vues de vidéos)

Les dimensions Vidéo et Jour affichent également les spectateurs uniques quotidiens, qui sont la moyenne des spectateurs uniques quotidiens pour la période sélectionnée. Ceci est calculé à l'aide de l'adresse IP et de l'agent utilisateur des visionneuses.

Sources de trafic

Lors de l'affichage des performances par source de trafic, les sources de trafic possibles sont les suivantes:

  • Parrainage - L'utilisateur a suivi un lien vers la vidéo
  • Direct - L'utilisateur a tapé le lien directement ou le référent n'a pas pu être déterminé
  • Recherche sécurisée - L'utilisateur s'est connecté avec un compte Google, a effectué une recherche et a cliqué sur un résultat (payant ou organique ; les informations de recherche ne sont pas incluses dans ce cas)
  • Recherche organique - L'utilisateur a effectué une recherche et a cliqué sur un résultat pour accéder à la vidéo
  • Recherche payante - L'utilisateur a effectué une recherche et a cliqué sur un résultat payant pour accéder à la vidéo

Utiliser des filtres

Les filtres peuvent être utilisés pour limiter les données qui apparaissent dans le rapport. Pour ajouter un filtre, procédez comme suit.

  1. Cliquez sur la pilule + Ajouter des filtres en haut du rapport.
  2. Les données peuvent être filtrées par balises, champs personnalisés, pays et types d'appareils. Configurez les critères de filtre à l'aide du panneau de filtres L'exemple suivant applique le filtre:
    • Tag = formation à la demande ET
    • Champ personnalisé Produit = Video Cloud ET
    • Pays = États-Unis
  3. Cliquez sur Appliquer les filtres.

Les filtres appliqués apparaîtront sous forme de comprimés en haut du rapport. Chaque pilule affichera un type de filtre et jusqu'à deux valeurs. Jusqu'à trois comprimés seront affichés.

Les pilules supplémentaires sont regroupées en une pilule de débordement et sont affichées dans une infobulle lors du survol.

Pour supprimer un filtre, cliquez sur un comprimé et effacez le filtre du panneau des filtres.

Remarques sur l'utilisation des filtres

  • Le type de flux n'est disponible que si Brightcove Live est activé sur le compte
  • La section des champs personnalisés sera désactivée si le compte n'est pas activé pour les champs personnalisés
  • Les filtres peuvent être ajoutés/supprimés à l'aide des boutons plus/moins situés à droite du type de filtre
  • Bien que les filtres semblent être appliqués, ils ne s'appliquent pas à toutes les sections du module Analytics
    • Les balises et les filtres de champs personnalisés ne sont pas appliqués lorsque vous cliquez sur une vidéo ou un lecteur spécifique pour afficher les détails de performance de la vidéo ou du lecteur (la seule exception étant les dimensions Lecteur par vidéo ou Jour)
    • Graphique d'engagement (graphique en haut de la page du rapport Engagement)
  • Le chargement des données Analytics peut prendre un peu plus de temps lors de l'application de filtres
  • Type de filtre
    • Tout - Doit inclure toutes les valeurs
    • Tout - peut inclure n'importe quelle valeur
    • Aucune - Ne doit inclure aucune des valeurs choisies

Exportation des données de

Les données du rapport peuvent être téléchargées sous forme de valeurs séparées par des virgules et de formats Excel. Notez que la fonctionnalité d'exportation enregistre les données directement à partir d'un appel d'API Analytics. Les données exportées ne sont pas manipulées de la même manière que les données de l'interface utilisateur et contiennent actuellement toutes les données sous-jacentes du rapport, et pas seulement le sous-ensemble de champs affiché par l'interface utilisateur. Certaines des différences entre ce qui est affiché dans l'interface utilisateur et les données exportées:

  • Différentes unités (minutes contre secondes)
  • Arrondi des nombres
  • L'exportation des données contient à la fois les mesures de performance et d'engagement, quel que soit votre emplacement dans l'interface utilisateur.
  • L'entrée Autre/Tiers sera exportée sans identifiant vidéo ni nom de vidéo associé

Le fait de disposer de toutes les données dans le format d'origine de l'API vous permet de manipuler les données d'une manière autre que celle fournie dans l'interface utilisateur.

Les boutons d'exportation situés au-dessus du graphique permettent d'exporter les données du graphique. Les boutons d'exportation situés au-dessus du tableau permettent d'exporter les données de la table. Notez que les rapports prennent en charge les feuilles de style imprimables. Par conséquent, l'impression d'un rapport ou l'enregistrement au format PDF générera une page imprimable.
