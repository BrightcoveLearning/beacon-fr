Présentation

Le processus de configuration comprend les tâches suivantes:

  • Créez quatre propriétés Google Analytics.
  • Créez un ensemble de dimensions personnalisées et une mesure personnalisée pour chaque propriété.
  • Créez quatre conteneurs Google Tag Manager.
  • Configurez Google Tag Manger pour chaque conteneur.
  • Configurez Firebase pour faciliter la collecte d'analyses avec iOS et Android.
  • Installez des modèles dans Google Data Studio pour afficher vos données d'analyse.

Outils nécessaires

Brightcove utilise divers outils pour mettre en œuvre la collecte, la manipulation et l'affichage des données d'analyse à l'aide de Google Analytics. Ces outils sont les suivants:

  • Google Analytics - Une plateforme d'analyse Web proposée par Google dans le cadre de sa marque Google Marketing Platform.
  • Google Tag Manager - Tout d'abord, comprenez que les balises sont des extraits de code qui sont ajoutés à un site pour collecter des informations et les envoyer à des tiers. Les étiquettes sont utilisées pour de nombreuses raisons, notamment le suivi du défilement, le suivi des soumissions de formulaires, la réalisation d'enquêtes, etc. Avec Google Tag Manager, vous pouvez gérer et mettre à jour les balises rapidement et prendre des décisions plus éclairées plus rapidement. Tag Manager s'intègre à tous les tags Google et tiers, et vous donne un contrôle total sur le moment et la manière dont vos tags sont déclenchés. «
  • Firebase - Firebase est une plate-forme développée par Google pour créer des applications mobiles et Web. Firebase inclut la base de données Firebase Realtime, qui est une base de données hébergée dans le cloud.
  • Google Data Studio - Google Data Studio est un outil gratuit qui transforme vos données en tableaux de bord et rapports informatifs, faciles à lire, faciles à partager et entièrement personnalisables. «

Obtenir des modèles et des fichiers de configuration

Il existe un certain nombre de fichiers/URL dont vous avez besoin pour terminer avec succès la configuration des analyses pour Brightcove {{site.product_short}}. Les fichiers/URL sont détaillés dans les puces suivantes. Vous devez contacter votre responsable d'intégration Brightcove pour obtenir ces fichiers/URL. Vous recevrez:

  • URL vers trois configurations prédéfinies du tableau de bord Google Analytics, une pour les appareils mobiles, le Web et la télévision connectée.
  • Quatre fichiers JSON qui configurent chacun des quatre conteneurs créés dans Google Tag Manager.
  • URL vers trois modèles Google Data Studio prédéfinis, un pour les appareils mobiles, le Web et la télévision connectée.

Les instructions d'installation/utilisation de chacun des fichiers/URL figurent dans les sections appropriées plus loin dans ce document.

Vous devrez obtenir ces fichiers/URL AVANT de commencer le processus de configuration.

Configuration de Google Analytics

Dans cette section du document, vous allez configurer Google Analytics, qui inclut les tâches suivantes:

  • Créez quatre propriétés Google Analytics.
  • Créer une vue mobile pour les propriétés iOS et Android
  • Créez un ensemble de dimensions personnalisées pour chaque propriété.
  • Créez une seule mesure personnalisée pour chaque propriété.
  • Importez trois configurations de tableau de bord créées par Brightcove, une pour les applications mobiles, Web et Smart TV.

Créez quatre propriétés

Vous devez créer ces quatre propriétés:

  • Beacon Android
  • Beacon iOS
  • Beacon STV (Smart TV)
  • Beacon Web

Les étapes suivantes vous guident tout au long de la création de la propriété Android Beacon. Vous devrez créer les autres en suivant le même processus.

  1. Connectez-vous à votre compte Google Analytics.
  2. Cliquez sur l'icône Admin (roue dentée) en bas à gauche de l'écran.
  3. En haut de la colonne du milieu, cliquez sur Créer une propriété.
  4. Dans la configuration de la propriété, indiquez le nom de la propriété, pour la première propriété, utilisez Beacon Android.
  5. Indiquez le fuseau horaire des rapports.
  6. Fournissez la devise.
  7. Cliquez sur le lien Afficher les options avancées.
  8. Activez l'option Créer une propriété Universal Analytics.
  9. Saisissez l'URL de votre site Web.
  10. Cliquez sur le bouton radio Créer une propriété Universal Analytics uniquement.
  11. Assurez-vous que les informations de votre propriété sont remplies de la manière suivante:
  12. Cliquez sur Suivant.
  13. Remplissez le formulaire d'information commerciale avec vos données.
  14. Cliquez sur Créer.
  15. Lisez tous les détails que vous souhaitez concernant votre nouvelle propriété.
  16. Cliquez sur la flèche retour, mise en surbrillance dans la capture d'écran suivante:
  17. Juste en dessous de la fenêtre Créer une propriété sur laquelle vous avez cliqué précédemment, cliquez sur la flèche vers le bas pour vous assurer que votre propriété a été correctement créée.
  18. Effectuez les étapes suivantes pour créer les trois autres propriétés requises:
    1. Beacon iOS
    2. Beacon STV
    3. Balise WEB

Créer une vue mobile pour iOS et Android

Pour les plateformes mobiles, iOS et Android, vous devez créer une vue mobile dans chacune de ces propriétés. Il n'est pas nécessaire de suivre les étapes de cette section si vous ne disposez pas d'applications mobiles.

  1. Cliquez sur le menu déroulant (surligné en jaune sur la capture d'écran) pour voir tous vos comptes, propriétés et vues Google Analytics.
  2. Dans la liste déroulante à trois panneaux, sélectionnez:
    • Votre compte Brightcove {{site.product_short}}
    • La propriété Beacon Android
    • La vue Données de tous les sites Web
  3. Cliquez sur l'icône Admin (roue dentée) en bas à gauche de l'écran.
  4. En haut de la troisième colonne, cliquez sur Créer une vue.
  5. Remplissez le formulaire suivant ces étapes:
    1. Cliquez sur le bouton de l'application mobile pour connaître le type de données à suivre.
    2. Donnez un nom à votre vue. Un nom logique serait All Mobile Data.
    3. Cliquez sur Créer une vue.
  6. Remplacez la vue par la vue mobile nouvellement créée.
  7. Suivez les mêmes étapes que dans cette section du document pour créer une vue mobile pour la propriété iOS.

Lorsque vous travaillez avec vos comptes, assurez-vous que lorsque vous sélectionnez un compte/propriété/affichez vos propriétés mobiles, utilisez la vue Toutes les données mobiles et les comptes Web et STV utilisent la valeur par défaut Toutes les données du site Web, comme montré ici:

Notez que les vues iOS et Android comporteront des vues Toutes les données du site Web, car elles sont créées par défaut lors de la création de la propriété.

Création de mesures et de dimensions personnalisées

Maintenant, pour chaque propriété, vous devez créer un ensemble de dimensions personnalisées et une métrique personnalisée.

Cette section sera organisée de la manière suivante:

  • Les étapes à suivre pour créer une seule dimension personnalisée seront expliquées.
  • Les étapes à suivre pour créer une seule mesure personnalisée seront expliquées.
  • Un tableau présente les dimensions requises pour les propriétés Beacon Android et Beacon iOS.
  • Un tableau présente les dimensions requises pour les propriétés Beacon STV et Beacon WEB.
  • Un tableau montre la seule mesure que doivent posséder les quatre propriétés.

Vous devrez ensuite créer les dimensions et la métrique pour chacune des quatre propriétés.

Création d'une dimension personnalisée

  1. Sélectionnez l'une de vos propriétés dans le sélecteur de compte/propriété/vue pour laquelle vous souhaitez créer les dimensions personnalisées.
  2. Cliquez sur l'icône Admin (roue dentée) en bas à gauche de l'écran.
  3. En bas de la colonne du milieu, sous Définitions personnalisées, cliquez sur Dimensions personnalisées
  4. Cliquez sur NOUVELLE DIMENSION PERSONNALISÉE.
  5. Pour le nom, saisissez le numéro de l'épisode.
  6. Dans la liste déroulante Scope, sélectionnez Hit.
  7. Laissez la dimension active.
  8. Cliquez sur Créer.
  9. Dans l'écran d'exemple de code, cliquez sur Terminé.
  10. Vous verrez la dimension que vous venez de créer dans une liste, comme illustré ci-dessous:
    Il est essentiel que l'index corresponde au nom de la dimension tel qu'indiqué dans les tableaux ci-dessous. Pour ce faire, il suffit de créer les dimensions dans l'ordre indiqué dans les tableaux.
  11. Pour créer une autre dimension, cliquez sur NOUVELLE DIMENSION PERSONNALISÉE.

Création d'une métrique personnalisée

  1. Sélectionnez l'une de vos propriétés dans le sélecteur de compte/propriété/vue pour laquelle vous souhaitez créer la mesure personnalisée.
  2. Cliquez sur l'icône Admin (roue dentée) en bas à gauche de l'écran.
  3. En bas de la colonne du milieu, sous Définitions personnalisées, cliquez sur Mesures personnalisées
  4. Cliquez sur NOUVELLE MESURE PERSONNALISÉE.
  5. Pour le Nom, saisissez Time Watched.
  6. Dans la liste déroulante Scope, sélectionnez Hit.
  7. Pour le type de formatage, sélectionnez Heure.
  8. Laissez la dimension active.
  9. Cliquez sur Créer.
  10. Dans l'écran d'exemple de code, cliquez sur Terminé.
  11. Vous verrez la mesure que vous venez de créer dans une liste.
Il s'agit de la seule mesure personnalisée que vous devrez créer pour chacune des quatre propriétés.

Configuration de Google Tag Manager

Dans cette section du document, vous allez effectuer les tâches suivantes:

  • Créez quatre conteneurs, un pour chaque propriété créée précédemment.
  • Importez les fichiers JSON de configuration qui vous ont été envoyés par e-mail dans la section Obtenir les modèles et les fichiers de configuration ci-dessus.
  • Saisissez votre identifiant de suivi, copié depuis votre compte Google Analytics, dans la variable du code de suivi de chaque conteneur.
  • Publiez vos conteneurs.

Création de conteneurs

  1. Consultez la documentation de Google pour configurer et installer Tag Manager.

  2. Dans Tag Manager, créez un compte ou utilisez un compte existant. Pour créer un nouveau compte, cliquez sur le bouton Créer un compte.

    Par défaut, un nouveau conteneur est créé pour un nouveau compte. Vous allez créer des conteneurs supplémentaires dans chaque compte.

  3. Entrez des valeurs pour la page de votre nouveau compte:

    • Nom du compte: Le nom de votre entreprise ou le nom du service.
    • Pays: Pays d'origine de votre entreprise.
    • Nom du conteneur: Beacon WEB Vous allez créer un conteneur pour chaque propriété.
    • Plateforme cible: Web
  4. Assurez-vous que les informations de votre contenant sont remplies de la manière suivante:
  5. Cliquez sur Créer.
  6. Cliquez sur OK pour fermer la boîte de dialogue qui s'ouvre.
  7. Dans l'espace de travail, cliquez sur Admin.
  8. Cliquez sur le signe plus pour ajouter un autre conteneur.
  9. Remplissez le formulaire et cliquez sur Créer pour créer le conteneur iOS.
  10. Assurez-vous que les quatre conteneurs suivants sont créés avec la plate-forme cible appropriée.
    Nom du conteneur Plateforme Target
    Beacon Android Android
    Beacon iOS iOS
    Beacon STV Web
    Balise WEB Web
  11. Dans l'espace de travail, utilisez le menu déroulant du compte en haut de la page pour vous assurer que tous les conteneurs ont été créés.
  12. À ce stade, il est essentiel de prendre une capture d'écran de ce menu déroulant. Les ingénieurs de Brightcove {{site.product_short}} ont besoin de ces données, à savoir les noms des conteneurs et l'ID GTM associé. Vous allez envoyer ce fichier à votre gestionnaire d'intégration avec les autres fichiers téléchargés, comme expliqué dans la section Publier chaque conteneur qui figure plus loin dans ce document. Fichiers de configuration d'importation

Effectuez les étapes suivantes pour importer les fichiers de configuration de vos conteneurs:

  1. Dans l'espace de travail, cliquez sur Admin.
  2. Vérifiez quel conteneur vous êtes en train de configurer, puis cliquez sur Importer un conteneur.
  3. Cliquez sur Choisir un fichier conteneur, puis sélectionnez le fichier JSON approprié pour ce conteneur.
  4. Pour Choisir un espace de travail, cliquez sur Existant, puis dans la boîte de dialogue qui s'ouvre, sélectionnez Espace de travail
  5. Pour Choisir une option d'importation, cliquez sur le bouton radio Remplacer.
  6. Assurez-vous de voir des informations sur les balises, les déclencheurs, les variables et les modèles.
  7. Cliquez sur Confirmer, et vous serez redirigé vers l'espace de travail. .
  8. Dans l'espace de travail, cliquez sur Tags, Triggers and Variables pour confirmer que l'importation a été réussi.
  9. Vous devez maintenant importer le bon fichier JSON pour chaque conteneur.

Saisissez l'ID de suivi

Vous allez maintenant copier à partir de Google Analytics l'ID de suivi de chaque propriété, puis le coller dans le conteneur correspondant.

  1. Connectez-vous à Google Analytics.
  2. Utilisez le sélecteur de compte/propriété en haut de la page pour choisir la propriété Android Beacon.
  3. Cliquez sur Admin.
  4. Dans la colonne Propriété, cliquez d'abord sur Informations de suivi, puis sur Code de suivi.
  5. Copiez le numéro de suivi.
  6. Revenir à la page Google Tag Manager.
  7. Assurez-vous de modifier le conteneur Android Beacon.
  8. Cliquez sur Variables dans la barre de navigation de gauche.
  9. Cliquez sur la variable Code de suivi.
  10. Collez l'ID de suivi copié depuis Google Analytics dans le champ du formulaire Valeur.
  11. Vous devez maintenant coller l'ID de suivi correspondant dans la variable Code de suivi de chaque conteneur.

Publier chaque conteneur

Plus loin dans ces étapes, vous publierez vos conteneurs. Pour iOS et Android, vous devrez télécharger les fichiers de configuration et demander à votre équipe de développement d'applications d'ajouter ces fichiers à l'application correspondante.

La dernière étape que vous devez effectuer dans Google Tag Manger consiste à soumettre chaque conteneur pour publication. Les étapes suivantes vous guident tout au long de ce processus:

  1. Revenez à votre espace de travail.
  2. Notez le conteneur dans lequel vous vous trouvez.
  3. En haut à droite de l'interface utilisateur, cliquez sur Envoyer.
  4. Assurez-vous que le bouton Publier et créer une version est sélectionné.
  5. Si vous le souhaitez, ajoutez un nom de version et une description de version.
  6. Cliquez sur Publier.
  7. Si vous n'avez pas fourni de nom de version ni de description, on vous le demande à nouveau. Ignorez si vous êtes sûr de ne pas les vouloir.
  8. Assurez-vous que les informations renvoyées concernant votre conteneur sont correctes.
  9. Pour les conteneurs iOS et Android: Cliquez sur Télécharger en haut à droite de l'interface utilisateur pour télécharger le fichier associé. Ce fichier devra être envoyé à votre responsable de l'intégration.
  10. Pour les conteneurs iOS et Android: Contactez votre responsable de l'intégration et envoyez-lui ces fichiers téléchargés, accompagnés de la capture d'écran de vos conteneurs et des identifiants correspondants. Les fichiers seront ajoutés à vos applications correspondantes.
  11. Assurez-vous que tous les conteneurs sont publiés.
Pour rappel, à ce stade, vous devez envoyer les fichiers téléchargés, ainsi que la capture d'écran montrant le nom de chaque conteneur et l'ID GTM associé, à votre gestionnaire d'intégration. Ces fichiers seront ensuite acheminés vers le personnel d'ingénierie de Brightcove {{site.product_short}} pour configurer vos applications.
