Les paramètres Live vous permettent de gérer les ardoises de remplissage et les informations d'identification Live.

Remarque: Les configurations publicitaires côté serveur utilisées par Brightcove Live sont configurées dans le cadre de l'administration des paramètres publicitaires côté serveur. Pour plus d'informations, consultez la section Configuration des paramètres publicitaires côté serveur.

Gestion des ardoises de remplissage

Les listes de remplissage sont du contenu qui peut être diffusé lorsqu'il y a des lacunes dans une pause publicitaire en direct. Les ardoises peuvent être utilisées pour fournir un message Revenez tout de suite-fr/ ou tout autre contenu que vous souhaitez afficher. Tous les formats d'entrée Zencoder sont pris en charge.

Remarque: Les ardoises de remplissage peuvent également être gérées à l'aide de l'API Live. Pour plus d'informations, consultez l'API Live de Brightcove: Insertion de publicités côté serveur (SSAI).

Pour ajouter une ardoise de remplissage, procédez comme suit.

  1. Ouvrez le module Admin.
  2. Dans le volet de navigation de gauche, cliquez sur Paramètres en direct.
    Remarque: Vous devez être administrateur de compte pour accéder au menu Admin.
  3. Cliquez sur + Ajouter une ardoise de remplissage.
  4. Saisissez un nom.
  5. Saisissez l'URL source de la liste.
  6. Cliquez sur Enregistrer. Le processus d'ingestion de l'ardoise va commencer.

Pour supprimer une ardoise de remplissage, cliquez sur le bouton Supprimer () en regard de la ardoise de remplissage.

Gestion des identifiants

Les informations d'identification sont utilisées à la fois par l'API Live et le module Live. Si vous créez un nouvel identifiant Live Video Cloud, vous devez d'abord créer un identifiant d'authentification API. L'ID client et le secret client créés seront utilisés lors de la création des informations d'identification Live Video Cloud. Lors de la création d'informations d'identification FTP, S3 ou RTMP, aucune information d'identification d'authentification API n'est nécessaire.

Si vous créez des informations d'identification S3, FTP ou RTMP, passez à l'étape 5. Pour créer un identifiant Live Video Cloud, commencez par créer un identifiant d'API.

  1. Ouvrez le module Admin.
    Remarque: Vous devez être administrateur de compte pour accéder au menu ADMIN.
  2. Dans le menu de navigation de gauche, cliquez sur Authentification API La page Authentification de l'API s'ouvre et affiche les inscriptions actuelles de vos clients.
  3. Cliquez sur Enregistrer une nouvelle application pour créer de nouvelles informations d'identification d'API. Les API suivantes doivent être sélectionnées.
    Remarque: Pour plus d'informations sur la création d'informations d'identification d'authentification d'API, consultez Gestion des informations d'authentification
  4. Lorsque les informations d'identification sont créées, enregistrez l'ID client et les valeurs secrètes du client. Ces informations seront nécessaires lors de la création des informations d'identification Live Video Cloud.
  5. Dans le volet de navigation de gauche, cliquez sur Paramètres en direct.
  6. Cliquez sur + Ajouter des informations d'identification.
  7. Saisissez une étiquette. Il s'agit du nom qui sera utilisé en interne pour faire référence aux informations d'identification.
  8. Sélectionnez un type:
    • FTP - Utilisé pour enregistrer la ressource VOD sur un emplacement FTP (utilisation de l'API)
    • RTMP: utilisé pour créer des informations d'identification RTMP à utiliser par l'encodeur
    • S3 - Utilisé pour enregistrer l'actif VOD sur un compte Amazon S3 (utilisation de l'API)
    • Video Cloud - Utilisé pour enregistrer la ressource VOD sur un compte Video Cloud (utilisation de l'API)
  9. Pour Public Key/Username, entrez ce qui suit en fonction du type d'informations d'identification:
    • FTP - Saisissez le nom d'utilisateur du site FTP
    • RTMP - Entrez le nom d'utilisateur que l'encodeur utilisera pour s'authentifier
    • S3 - Entrez le nom d'utilisateur du compte S3
    • Video Cloud: saisissez l'ID client généré pour les informations d'identification d'authentification API
  10. Pour Clé privée/Mot de passe, entrez ce qui suit en fonction du type d'informations d'identification:
    • FTP - Entrez le mot de passe du site FTP
    • RTMP - Entrez le mot de passe que l'encodeur utilisera pour s'authentifier
    • S3 - Entrez le mot de passe du compte S3
    • Video Cloud: saisissez le secret client généré pour les informations d'identification d'authentification API
  11. Cliquez sur Enregistrer.

Les informations d'identification apparaîtront dans la liste des informations d'identification. Pour supprimer une information d'identification, cliquez sur le bouton Supprimer () en regard des informations d'identification.
