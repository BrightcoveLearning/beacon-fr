Création d'inscriptions de clients

La page Authentification de l'API est utilisée pour créer de nouvelles inscriptions de clients. Lors de la création d'inscriptions, vous pouvez sélectionner le service et les rôles qui seront activés avec l'enregistrement.

Pour créer une nouvelle inscription client pour votre compte:

  1. Ouvrez le module Admin.
  2. Cliquez sur le lien Authentification API. La page Authentification de l'API s'ouvre et affiche les inscriptions actuelles de vos clients.
    Remarque: Seules les inscriptions des clients pour le compte auquel vous êtes connecté s'afficheront.
  3. Cliquez sur Enregistrer une nouvelle application.
  4. Entrez un nom et une description pour l'enregistrement du client.
  5. Sélectionnez un ou plusieurs de vos comptes pour l'autorisation.
  6. Sélectionnez la ou les API et les autorisations à activer pour cet enregistrement.
    Remarques:
  7. Pour l'intégration de DFP/Google Ad Manager, les autorisations d'API nécessaires sont CMS Video Read.
  8. La sélection d'un grand nombre de comptes et d'API peut entraîner une taille d'en-tête HTTP importante et non valide. La taille maximale d'en-tête acceptée par notre serveur est de 8 Ko, ce qui signifie que vous devez sélectionner 30 comptes ou moins pour appliquer les informations d'identification. Pour des raisons de sécurité, nous vous recommandons de limiter la portée des autorisations pour un ensemble d'informations d'identification au plus petit ensemble de comptes et d'opérations nécessaires.
  9. Cliquez sur Enregistrer.
  10. Un ID client et un secret client vous seront renvoyés.
    Remarque: Copiez le secret client et stockez-le dans un emplacement sécurisé car le secret du client ne s'affichera plus.

Gérer les inscriptions des clients

La page Authentification de l'API peut être utilisée pour afficher et gérer les inscriptions des clients. Cliquez sur le lien ADMIN dans l'en-tête Studio, puis cliquez sur Authentification API. Une liste de vos inscriptions en cours s'affichera.

Pour afficher les détails de l'inscription, cliquez sur le nom. À partir de là, vous pouvez:

  • Modifier le nom
  • Copiez l'ID client
  • Ajouter des comptes pour autorisation
  • Sélectionnez des API supplémentaires à activer
  • Supprimer les informations d'identification

Cliquez sur Enregistrer pour enregistrer vos modifications.

Notez que si des informations d'identification sont créées pour plusieurs comptes, les utilisateurs ne verront ces informations d'identification que s'ils ont accès à tous ces comptes. Par exemple, l'utilisateur 1 crée des informations d'identification pour les comptes A et B. L'utilisateur 2 se connecte et n'a accès qu'au compte A. L'utilisateur 2 ne verra pas les informations d'identification car il n'a pas accès au compte B.

Mise à jour des inscriptions

Bien que vous ne puissiez pas récupérer le secret client pour les informations d'identification existantes, vous pouvez cliquer sur un nom d'identification existant, puis ajouter ou supprimer à la fois des autorisations et des comptes.

Suppression des enregistrements clients

Pour supprimer l'enregistrement d'un client, cliquez sur l'icône de suppression () en regard de l'enregistrement ou modifiez l'enregistrement et cliquez sur Supprimer l'enregistrement. La suppression d'un enregistrement le désactive définitivement et ne peut pas être annulé.

Travailler avec DFP

Si vous disposez actuellement de plusieurs comptes {{site.product_short}} configurés comme sources de contenu dans DFP, deux options s'offrent à vous:

  1. Vous pouvez créer un ID client et un secret client uniques pour chaque compte {{site.product_short}} configuré en tant que source de contenu dans DFP.™ Par exemple, si vous avez 10 comptes {{site.product_short}}, vous aurez 10 applications nommées DFP dans vos paramètres d'authentification API dans {{site.product_short}}.
  2. Vous pouvez créer un identifiant client et un code secret client uniques pour tous les comptes {{site.product_short}} qui ont été configurés comme sources de contenu dans DFP. Par exemple, si vous avez 10 comptes {{site.product_short}}, vous aurez 1 application nommée DFP dans vos paramètres d'authentification API dans {{site.product_short}}.

Que vous ayez choisi l'option #1 ou #2, vous devez fournir un ID de compte unique pour chaque source de contenu que vous avez configurée dans DFP. Ainsi, vous ne réduisez pas le nombre de sources de contenu actuellement configurées dans DFP ; vous remplacez simplement le jeton de lecteur par l'ID client et le secret client, qui peuvent être identiques dans toutes les sources de contenu ou uniques pour chacune d'entre elles.

L'option #1 prend plus de temps à l'avance. Toutefois, si l'ID client et le secret client sont les mêmes pour toutes les sources de contenu (comme pour l'option #2), il sera difficile d'ajouter une nouvelle source de contenu car vous devrez générer un nouvel ID client et un nouveau secret client et mettre à jour toutes les sources de contenu dans DFP. Brightcove recommande donc l'option #1.
