Champs pour les profils Live Ingest personnalisés

Cette rubrique fournit une référence pour les champs utilisés dans les profils d'acquisition Live.

Champs du profil

Propriété Type Description Par défaut
account_id entier L'identifiant du compte Video Cloud aucune
nom chaîne Le nom du profil - doit être unique dans le compte aucune
nom_affichage chaîne Le nom d'affichage du profil à afficher sur la page d'administration de Studio - peut être identique au nom, mais doit être descriptif pour aider les utilisateurs à comprendre l'utilisation prévue pour le profil aucune
description chaîne La description du profil aucune
modèle_version entier Réglez cette valeur sur 1 pour que le profil apparaisse dans le module Live. aucune
restitutions gamme Tableau d'objets de rendu []
paquets gamme Tableau de packages DRM - actuellement, DRM n'est pas pris en charge, il doit donc toujours s'agir d'un tableau vide []

Champs de format associé

Les tableaux ci-dessous présentent les champs qui peuvent être utilisés dans le corps de la demande pour la mise à jour des profils d'ingestion en direct.

Propriété Type Description Par défaut
audio_bitrate entier débit audio en kbit/s (min: 16, max: 1024) source de correspondance
codec audio chaîne Codec audio - AAC | MP3 Déterminé par le format ou le profil, dans la plupart des cas, AAC sera le meilleur
decoder_bitrate_cap entier en kbps, le débit binaire maximal transmis au décodeur via un tampon aucune
decoder_buffer_size entier en kbps, la taille du tampon envoyé au décodeur lors de l'utilisation d'un bitrate_cap aucune
forcéd_keyframe_rate nombre Force une image-clé toutes les X images, mais autorise toujours des images-clés supplémentaires pour la découpe de scène, etc. aucune
format chaîne Format vidéo - TS, MP4, FLV aucune
h264_profile chaîne H.264 possède trois profils couramment utilisés: Niveau de référence (le plus bas), principal et élevé. de base
hls_optimized_ts booléen Segments temporels optimisés pour HLS - s'applique uniquement à la sortie HLS aucune
keyframe_interval entier Nombre maximal d'images entre les images-clés 250
keyframe_rate flotte Nombre maximal d'images-clés par seconde keyframe_interval de 250
étiquette chaîne Type d'image pour les rendus d'image - affiche ou miniature aucune
live_stream booléen s'il s'agit d'un flux vidéo en direct aucune
live_dvr_sliding_window_duration entier Durée du flux pour rester disponible pour la diffusion LiveDVR en quelques secondes 300
max_video_bitrate entier en Kbits/s, 123 = 123 kbps - h264 uniquement aucune
type_média chaîne vidéo ou audio aucune
video_bitrate entier en kbits/s, 123 = 123 kbps (min: 64, max: 10 000) aucune
code_vidéo chaîne par exemple, h264, ts Déterminé par le format ou le profil
largeur entier largeur du cadre en pixels aucune
la taille entier hauteur d'image en pixels aucune
