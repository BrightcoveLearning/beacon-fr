Présentation

Dans ce document, vous trouverez un aperçu général de l'ensemble du processus de mise en œuvre de Brightcove Beacon. Il mettra l'accent sur le fait qu'en tant que client, vous devez effectuer un certain nombre de tâches avant de rencontrer un représentant de Brightcove pour créer les applications de votre appareil.

Aperçu

    Dans cette section, les phases sont présentées, brièvement définies et le résultat de la phase est indiqué. Plus loin dans le document, des listes de contrôle, le cas échéant, sont fournies pour vous aider à vous préparer à chaque phase.

  1. Phase 1: Préparation à la création de Brightcove Beacon Experience

    Plus tard dans le processus de mise en œuvre de Brightcove Beacon (phase 4), vous créerez l'expérience Brightcove Beacon. Pour ce faire, vous travaillerez avec un représentant Brightcove. À ce stade, vous aurez besoin de certains artefacts de conception prêts, ainsi que des décisions prises sur les plateformes sur lesquelles vous souhaitez déployer, la stratégie de monétisation, la stratégie publicitaire, etc.

    RÉSULTAT FINAL: Ensemble de ressources et de stratégies définies pour être utilisées lors d'une réunion avec un représentant de Brightcove pour créer l'expérience Brightcove Beacon.

  2. Phase 2: Établir des relations avec des fournisseurs tiers

    Il existe de nombreuses relations avec des fournisseurs tiers que vous devez établir avant de rencontrer un représentant de Brightcove pour créer l'expérience Brightcove Beacon. Cela inclut de contacter les fournisseurs des plateformes sur lesquelles vous choisissez de déployer une application, un fournisseur de serveurs publicitaires, Stripe pour les paiements en ligne si vous lancez des applications Web et/ou Smart TV, etc. Il est TRÈS IMPORTANT que ce processus démarre le plus tôt possible afin de ne pas entraîner de retards au moment de la soumission de l'application de l'appareil.

    RÉSULTAT FINAL: Comptes créés auprès de tous les fournisseurs tiers souhaités pour une utilisation lors d'une réunion avec un représentant de Brightcove pour créer l'expérience Brightcove Beacon.

  3. Phase 3: Créez la conception de votre application

    Deux parties de la conception de votre application sont réalisées ici. La première consiste à planifier les vidéos qui seront mises à la disposition des clients et comment les regrouper et les commander. Essentiellement, votre architecture d'information. La deuxième partie consiste à choisir l'apparence des écrans spécifiques de votre application, en fonction d'un certain nombre de choix.

    RÉSULTAT FINAL: Deux feuilles de calcul, créées lorsque vous parcourez les documents appropriés, sont remplies et prêtes à être soumises au personnel de Brightcove lorsque l'expérience Brightcove Beacon est prête à être créée.

  4. Phase 4: Intégrez ou ajoutez du contenu et configurez votre expérience Brightcove Beacon

    Ici, vous allez utiliser la console d'administration Brightcove Beacon pour configurer votre expérience. Ici, vous allez importer des ressources multimédias de Video Cloud et ajouter d'autres contenus. La configuration de la monétisation et de la mise en page est également effectuée. À ce stade, il n'est pas nécessaire d'ajouter TOUT le contenu, mais au moins un contenu représentatif pour des tests approfondis est nécessaire.

    RÉSULTAT FINAL: Le contenu est ajouté et la configuration est effectuée via Brightcove Beacon.

  5. Phase 5: Créez des applications d'appareil à partir de votre expérience Brightcove Beacon

    Vous travaillerez avec un représentant de Brightcove pour créer l'expérience Brightcove Beacon. Le travail effectué dans les phases 1 à 4 est utilisé pour créer l'expérience. À partir de Brightcove Beacon Experience, les applications de l'appareil sont créées. À ce stade, des tests d'acceptation des utilisateurs sont également effectués pour s'assurer que les applications fonctionnent comme souhaité.

    RÉSULTAT FINAL: Les fichiers de l'application de l'appareil sont créés, testés et disponibles pour être soumis aux vendeurs/boutiques souhaités.

  6. Phase 6: Soumettez votre (vos) application (s) pour appareil (s) aux boutiques correspondantes

    À ce stade, vous avez les applications pour les plateformes souhaitées en main, ainsi que la configuration effectuée, de sorte que vous pouvez maintenant soumettre les candidatures.

    RÉSULTAT FINAL: Les applications de l'appareil sont soumises de manière appropriée et les utilisateurs commencent à utiliser les applications après approbation du fournisseur.

Timing des phases

Notez que les phases 1 et 2 peuvent et doivent être exécutées simultanément.

Bien entendu, une fois les applications de l'appareil lancées, vous pouvez continuer à configurer et à ajouter du contenu à Brightcove Beacon sans avoir à soumettre à nouveau les applications de votre appareil à leurs magasins respectifs.

Détails - Phase 1: Préparation à la création de Brightcove Beacon Experience

Voici les tâches à effectuer pour la phase 1:

Déterminer les plateformes d'applications requises pour les appareils et les plateformes prioritaires
Déterminer la stratégie de monétisation
Déterminer si des services d'authentification utilisateur personnalisés sont nécessaires
Obtenez les couleurs et logos de votre marque pour développer l'expérience et les spécifications de votre appareil
Assister à une formation Video Cloud: webinaire ou formation à la demande
Intégrez des ressources multimédias dans Video Cloud ; vous aurez besoin d'un minimum de 40 vidéos et de créer de 3 à 5 listes de lecteur utilisant ces vidéos
Déterminer les exigences relatives
Déterminer les géolocalisations restreintes, le cas échéant

Détails - Phase 2: Établir des relations avec des fournisseurs tiers

Voici les tâches à effectuer pour la phase 2:

Déterminer la stratégie publicitaire et établir une relation avec le fournisseur approprié
Établissez une relation avec Stripe pour les paiements en ligne, si vous lancez des applications Web ou pour appareils Smart TV D'autres plateformes disposent d'un processus de paiement intégré.
Configurez les comptes de la boutique et confirmez les informations Les fournisseurs peuvent inclure, mais sans s'y limiter:
  • Plateforme Apple Developer
  • Plateforme Google Developer
  • Plateforme des développeurs Amazon
  • Plateforme de développement Roku
  • Autres plateformes de développement (par exemple Samsung)
Consultez la section Soumission des applications de l'appareil aux magasins pour plus d'informations et les URL correspondantes pour démarrer ce processus.

Détails - Phase 3: Créez la conception de votre application

Vous devez réfléchir à la manière dont vous souhaitez que les applications de votre appareil apparaissent aux yeux de vos clients. Au cours de cette phase, vous devez examiner les deux documents:

  • Création de votre conception d'application - Architecture de l'information
  • Création de la conception de votre application - Options de disposition

Après avoir examiné ces deux documents, vous disposerez de deux feuilles de calcul, une créée dans chaque document, que vous remettrez au personnel de Brightcove au moment de créer l'expérience Brightcove Beacon (phase 4). Les informations contenues dans les deux feuilles de calcul détaillent les vidéos présentées aux utilisateurs, y compris la manière dont elles sont regroupées et classées. Des dispositions d'écran spécifiques sont également sélectionnées.

Détails - Phase 4: Intégrez ou ajoutez du contenu et configurez votre expérience Brightcove Beacon

Vous avez peut-être effectué d'autres tests et configurations, mais dans cette phase, vous souhaitez terminer la configuration. Il n'est pas nécessaire d'ajouter TOUT le contenu, mais à des fins de test, assurez-vous d'avoir un contenu représentatif pour tous les types de médias que vous utiliserez. Vous utiliserez la console d'administration Brightcove Beacon Experience pour effectuer une configuration plus approfondie.

Détails - Phase 5: Créez des applications d'appareil à partir de votre expérience Brightcove Beacon

Cette phase se déroule avec l'aide du personnel de Brightcove. Les ressources, les relations avec les fournisseurs, les stratégies définies, etc. des trois premières phases sont utilisées par le représentant de Brightcove pour créer et effectuer la configuration initiale de Brightcove Beacon Experience. À partir de cette collaboration, les fichiers d'application de l'appareil sont générés à partir de Brightcove Beacon. Au cours de cette phase également, vous pouvez effectuer des tests d'acceptation sur les applications de l'appareil.

Détails - Phase 6: Soumettez votre (vos) application (s) pour appareil (s) aux boutiques correspondantes

Pour chaque fournisseur pour lequel vous disposez d'une application pour appareil, vous devez soumettre l'application à la boutique correspondante. Au cours de la phase 2, vous avez précédemment établi une relation avec la plateforme du fournisseur. Dans cette phase, vous soumettez donc l'application pour commencer le processus d'approbation.™ Dans le document Soumission des applications pour appareils aux magasins, vous trouverez des étapes spécifiques et des liens vers la documentation appropriée du fournisseur pour vous aider dans le processus de soumission.
